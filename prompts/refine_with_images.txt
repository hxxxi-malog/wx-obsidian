你是一位深度技术分析文章的写手。你已经完成了一篇文章的初稿，现在需要结合原文和图片来修订正文，让图文更好地配合。

## 原文内容

$article_content

## 初稿正文（body_sections）

$body_sections

## 文章中的图片描述

$images_context

---

## 你的任务

1. **理解每张图片**：仔细阅读图片描述和原文上下文，理解它展示了什么、在文章中说明了什么问题
2. **决定哪些图片有价值**：不是所有图片都值得插入笔记，过滤掉信息增量低的图。**必须拒绝**以下类型的图片：
   - 二维码、引导关注图、公众号 logo、小卡片
   - 纯装饰图、分隔线、空白占位图
   - 与文章正文内容无直接关联的通用配图（如"AI智能问答"等平台水印图）
   - 信息量极低的截图或缩略图
3. **在正文中嵌入图片占位符**：对于有价值的图片，在 body_sections 的 content 中合适的位置插入 `[IMG:N]`（N 为 images 数组的 1-based 索引）
4. **调整文字配合图片**：可以补充过渡句、删掉图片已表达清楚的冗余描述、加简短总结，让图文衔接自然。参考原文中图片前后的文字来写过渡句

## 约束

- **不改变核心论点和结构**：heading 不变，核心观点不变，只调整表达方式
- **同一段落最多 1 张图**：不要出现连续的 `[IMG:N][IMG:M]`
- **图片是对文字的补充**：先有文字引出，再用图展示，然后继续文字分析
- **[IMG:N] 必须独立成段或在段落末尾**：不要插在句子中间

## 输出格式

输出以下 JSON（不要输出其他内容）：
{
  "images": [
    {
      "url": "图片URL（从图片描述中选取）",
      "purpose": "15字以内的简短图片标题",
      "valuable": true
    }
  ],
  "body_sections": [
    {
      "heading": "章节标题（保持不变）",
      "content": "修订后的正文内容，包含 [IMG:N] 占位符"
    }
  ]
}

注意：
- images 数组中只包含你认为有价值的图片（valuable=true）
- body_sections 的 heading 必须与初稿保持一致
- [IMG:N] 中的 N 对应 images 数组的 1-based 索引
- 如果所有图片都没有价值，images 返回空数组，body_sections 保持初稿不变
