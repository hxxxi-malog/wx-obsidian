# 文章主体写作风格

## 写作核心哲学

**拓展思路，而非罗列信息。**

每篇文章的目标不是把原文内容"翻译"成笔记，而是帮读者理解：这篇文章想让人产生什么样的思考？它打破了哪些常见认知？它提供了什么新的视角？

具体原则：
- **揭示本质**：不停留在"是什么"，要追问"为什么是这样"和"为什么不是那样"
- **对比认知**：用"并非X，而是Y"的对比句式，打破读者的默认假设
- **推导链条**：展示从前提到结论的推理过程，而非直接扔结论
- **落地价值**：每个核心洞察都要回答"所以呢？对我的实践有什么启发？"

## 开头段落

用**一段话**构建全文的"论点炸弹"，必须包含：
- 分析对象名称（加粗）
- 核心洞察（用"并非X，而是Y"对比句式，打破常见认知）
- 解决的核心痛点
- 最终价值/启发

**写作要求**：信息密度要高，一段话要让读者立刻理解"这篇文章为什么值得读"。

**示例：**
> Claude Code v2.1.88的核心竞争力，并非单点技术突破，而是通过**12层渐进式工程包装（Progressive Harness）**，将一个极简的Agent核心循环，逐层升级为安全、高效、可扩展的工业级自主编码代理。它用工程化设计解决了AI Agent落地的核心痛点：上下文污染、行为不可控、安全风险、token效率低下，重新定义了工业级Agent的设计标准。

## 章节结构

### 一级标题（一、二、三、四...）
4-6 个章节，典型结构（可根据文章内容灵活调整）：

| 章节 | 核心职责 | 适用场景 |
| --- | --- | --- |
| 一、核心概念与背景 | 定义关键概念，厘清易混淆术语，建立讨论基础 | 几乎所有文章都需要 |
| 二、问题与矛盾分析 | 揭示核心矛盾、痛点、固有挑战 | 偏分析型文章 |
| 三、核心机制/架构拆解 | 文章主体，深度拆解技术细节 | 几乎所有文章都需要 |
| 四、核心认知升级 | 提炼哲学、原则、思维模型 | 偏深度分析文章 |
| 五、落地指南与总结 | 行动建议、全景总结、回扣开头 | 几乎所有文章都需要 |

**灵活变体**（根据文章类型选择）：
- **推导型**（如第一性原理文章）：公理定义 → 假设审视 → 最佳实践推导 → 落地框架
- **实践型**（如Harness落地文章）：概念厘清 → 从0到1演进路径 → 踩坑与修复 → 全景总结
- **拆解型**（如Claude Code文章）：架构全景 → 核心机制逐层拆解 → 未来路线图 → 工程哲学

### 二级标题
每个一级章节下 2-4 个二级标题，用 `### （一）` `### （二）` 编号（中文括号数字），标题要概括该节的核心论点，而非仅仅描述内容。

### 三级标题
用 `1. 2. 3.` 编号，展开二级标题下的具体要点。

## 表格使用

**每个核心章节至少一个表格**，表格是拓展思路的核心工具。

### 表格的四种核心用法

| 用法 | 适用场景 | 列数建议 |
| --- | --- | --- |
| 方案/技术对比 | 选型决策、路线对比、优劣分析 | 3-4列（方案/优势/问题/结论） |
| 要素归纳 | 概念厘清、模块定位、职责划分 | 3-4列（概念/回答的问题/定位/补充说明） |
| 分层说明 | 架构层级、演进阶段、优先级 | 3-4列（层级/名称/核心能力/工程洞察） |
| 双面分析 | AI的改善与新问题、变与不变 | 3-4列（维度/原有问题/AI改善/AI新问题） |

### 表格写作要求
- 表格前用一句话引出，说明为什么需要这个表格
- 每个单元格信息密度要高，不要写空泛的描述
- 对比类表格最后一列通常是"结论"或"关键洞察"

**示例（选型对比表）：**

> 针对单Agent在复杂需求下的失稳问题，文章对比了三条技术路线：

| 技术路线 | 核心优势 | 核心问题 | 最终选型结果 |
| --- | --- | --- | --- |
| 继续强化单Agent | 成本最低、改造最少 | 角色冲突愈发严重，长链路任务极易失稳 | 仅作为早期阶段有效 |
| 去中心化协作 | 足够灵活，应变能力强 | 行为路径不稳定、责任边界不清、极难长期维护 | 明确放弃，不适合工程长期维护 |
| 结构化调度 | 流程清晰、可控性强、结果可审计 | 前期设计成本更高、灵活性相对较弱 | 最终选定主路线 |

## 图片使用

**图片由系统自动插入**，通过两轮 LLM 处理：第一轮生成纯文本笔记，第二轮结合图片描述在合适位置嵌入 `[IMG:N]` 占位符。正文中无需使用 `![](url)` 标记，也不要输出 `images` 字段。

### 正文写作与图片配合

1. **为图片留出空间**：在分析图表、架构、流程等内容时，写出让图片自然出现的上下文
2. **文字引出图片**：先用文字描述要说明的问题，图片是对文字的视觉补充
3. **同一段落最多 1 张图**，不要连续插入图片

## 核心洞察与重点标记

### "核心洞察"标记
文章中最关键的结论，用 `**核心洞察**：` 前缀引出，让读者一眼抓住重点。

**示例：**
> **核心洞察**：所有复杂性都被外推到两个地方——工具层和包装层，核心循环永不改变，完美践行了开闭原则。

### 对比句式（高频使用）
- "并非A，而是B"
- "不是A，而是B"
- "本质是X" / "核心是Y" / "关键在于Z"
- "表面上是A，实际上是B"

### 概念首次出现
核心概念首次出现时加粗，中英文并存保留英文，如 **渐进式工程包装（Progressive Harness）**。

## 常用句式与术语

**高频句式**（用于拓展思路）：
- "X的核心是Y"
- "本质是Z"
- "解决了W的核心痛点"
- "从X到Y的完整路径"
- "N大核心拼图/N层渐进式机制"
- "这是全文最核心的洞察/落地指导"
- "打破了XX的常见认知"
- "提供了新的视角：..."

**高频术语**：核心、本质、落地、拆解、解构、范式、工程化、闭环、痛点、演进、第一性原理、公理、约束优化

**避免**：口语化、过度简化、无依据断言、"综上所述"等八股结尾

## 结尾结构

### 核心认知/哲学章节
提炼 3-7 条核心原则，每条一句话概括，用编号列表。这是文章"拓展思路"的核心价值所在。

**示例：**
1. **"One Loop & Bash is all you need"**：核心循环极简，复杂性全部推到工具层和包装层
2. **"Adding a tool = adding one handler"**：完美践行开闭原则，新增功能不修改核心循环
3. **"An agent without a plan drifts"**：Planning机制是任务完成率翻倍的"免费午餐"

### 行动指南
3-5 条可落地建议，每条要具体、可执行，不要写空泛建议。可以用优先级标注（P0/P1/P2）。

**示例：**
| 优先级 | 核心动作 | 启发 |
| --- | --- | --- |
| P0 | 实现基础 Agent Loop + Bash Tool | 最小可行产品，先跑通核心闭环 |
| P0 | 添加 Planning 机制 | 零成本实现任务完成率翻倍 |
| P1 | 实现 Sub-Agent + 上下文隔离 | 解决上下文污染的核心痛点 |

### 总结段落
回扣开头论点，展望未来方向。不要用"综上所述"，要用有力量的收尾。

**示例：**
> Claude Code v2.1.88的优秀，从来不是因为Claude模型本身的能力碾压，而是它用极致的工程化设计，把大模型的能力封装成了安全、可控、高效的工业级产品。它的12层渐进式包装、安全优先的工具设计、按需注入的知识体系、透明的上下文管理，为所有AI Agent开发者提供了可复用的落地范式。
