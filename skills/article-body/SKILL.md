# 文章主体写作风格

## 开头段落

用**一段话**概括全文核心论点，必须包含：
- 分析对象名称（加粗）
- 核心洞察（用"并非X，而是Y"对比句式）
- 解决的核心痛点
- 最终价值

**示例：**
> Claude Code v2.1.88的核心竞争力，并非单点技术突破，而是通过**12层渐进式工程包装**，将一个极简的Agent核心循环，逐层升级为安全、高效、可扩展的工业级自主编码代理。

## 章节结构

### 一级标题（一、二、三...）
4-6 个章节，典型结构：
1. 核心概念定义与背景
2. 问题与矛盾分析
3. 核心机制/架构拆解（主体）
4. 核心认知升级/哲学
5. 总结与行动指南

### 二级标题（### 1. ### 2.）
每个一级章节下 2-4 个，标题概括该节核心内容。

### 三级标题（#### (1) #### (2)）
展开二级标题下的具体要点。

## 表格使用

**每个核心章节至少一个表格**，用途：
- 方案/技术对比（优劣、选型）
- 要素归纳（组成、定位、作用）
- 分层说明（层级、职责）
- 阶段演进（步骤、目标）

表格前用一句话引出。示例：

> 文章首先明确了Harness体系中6个最易混淆的核心概念：

| 核心概念 | 核心回答的问题 | 工程定位 |
| --- | --- | --- |
| Rule | 什么事绝对不能乱来 | 工程基础规矩、红线 |
| Skill | 这件事具体应该怎么做 | 固定动作的标准操作手册 |

## 重点标记

- **核心概念**：首次出现加粗，中英文并存保留英文
- **核心洞察**：用 `**核心洞察**：` 前缀引出关键结论
- **对比句式**："并非A，而是B" / "不是A，而是B"
- **本质定义**："本质是X" / "核心是Y" / "关键在于Z"

## 代码与图示

- 代码片段：仅说明核心机制时使用，10-20行，带注释
- 图片：`![描述](URL)`，前后用文字说明
- ASCII 图表：适合架构层次、流程关系

## 语言风格

**常用句式：**
- "X的核心是Y"
- "本质是Z"
- "解决了W的核心痛点"
- "从X到Y的完整路径"
- "N大核心拼图/N层渐进式机制"

**常用术语**：核心、本质、落地、拆解、解构、范式、工程化、闭环、痛点、演进

**避免**：口语化、过度简化、无依据断言

## 结尾

### 核心认知/哲学章节
提炼 3-7 条核心原则，每条一句话概括。

### 行动指南
3-5 条可落地建议。

### 总结段落
回扣开头论点，展望未来方向。

### 参考来源
文末附原文链接格式：
```
> 参考文章：[标题](URL)
```
